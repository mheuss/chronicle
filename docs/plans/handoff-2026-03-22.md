## Decisions Made

- **screencapturekit-rs over raw objc2:** Safe Rust bindings eliminate hundreds of lines of unsafe FFI. v1.5.4, well-maintained.
- **Channel-based frame delivery:** SCK callbacks push frames over tokio mpsc. try_send to never block SCK thread. Channel closure = engine death signal.
- **Developer guide as implementation deliverable:** Each feature branch includes a guide in docs/guides/ as the final task. To be formalized as an SOP workflow rule.
- **User-configurable idle capture interval:** HEU-254 will store `idle_capture_interval_ms` in the config table (default 10s) so users monitoring long-running shell commands can keep a faster capture rate.

## Raw Learnings

- `screencapturekit-rs` needs `DYLD_LIBRARY_PATH="/Library/Developer/CommandLineTools/usr/lib/swift-5.5/macosx"` for test binaries to find `libswift_Concurrency.dylib`. The swift-5.5 path may vary by system.
- macOS path canonicalization gotcha: `/var/folders/...` resolves to `/private/var/folders/...`. Orphan sweep must canonicalize paths to avoid deleting tracked files. Fixed by canonicalizing at allocation time.
- `cargo test` without DYLD_LIBRARY_PATH crashes with SIGABRT for any crate depending on screencapturekit. Unit tests that don't touch SCK types at runtime work fine.
- Linear auto-completes parent issues when a PR with the parent's branch name is merged. Need to manually reopen if sub-issues remain.
